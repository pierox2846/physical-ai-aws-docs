# NVIDIA Cosmos WFM on Amazon EKS

이 페이지는 Cosmos WFM 워크로드를 EKS에 올릴 때의 **운영 관점**을 정리합니다. Cosmos 세부 설치는 NVIDIA 공식 문서를 우선으로 봅니다.

### 목표

* GPU 노드 풀을 분리해서 운영합니다.
* 데이터는 S3 중심으로 저장합니다.
* 학습/추론 잡을 **잡 단위**로 스케줄합니다.

### 권장 구성(요약)

* 오케스트레이션: Amazon EKS.
* 노드: GPU 인스턴스 노드 그룹.
* 컨테이너 이미지: Amazon ECR.
* 데이터: Amazon S3 + (필요 시) 고성능 파일시스템.
* 관측: 로그/메트릭 표준화.

### 체크리스트

* GPU 드라이버/런타임을 어떻게 표준화할지.
* 대용량 데이터 I/O 병목을 어떻게 줄일지.
* 재현성을 위해 컨테이너/파라미터/데이터 버전을 어떻게 남길지.

{% hint style="info" %}
WFM은 실험 단계에서 변경이 잦습니다. 처음부터 ‘완전 자동화’보다 ‘반복 가능하게’에 집중하세요.
{% endhint %}
