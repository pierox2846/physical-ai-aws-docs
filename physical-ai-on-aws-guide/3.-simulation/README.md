# 3. 시뮬레이션 (Simulation)

시뮬레이션은 2가지를 위해 씁니다. **데이터를 늘리고**, **배포 전 실패를 찾습니다**.

### Simulation이 해결하는 것

* 희귀 케이스 생성.
* 안전 사고 재현.
* 카메라/조명/마찰 같은 편차 주입.
* 라벨 비용 절감(정답을 시뮬이 제공).

### 설계 포인트

* 시뮬 시나리오를 코드로 버전 관리합니다.
* 랜덤 시드와 파라미터를 기록합니다.
* 학습용과 검증용 시나리오를 분리합니다.

### AWS에서 운영 관점 체크

* 배치 시뮬을 어떻게 스케일할지.
* 결과(비디오/라벨/로그)를 어떻게 저장할지.
* 렌더링 워크로드를 어디서 돌릴지.

{% hint style="info" %}
시뮬은 ‘더 예쁘게’가 목표가 아닙니다. 현실에서 깨지는 지점을 더 빨리 찾는 게 목표입니다.
{% endhint %}
