# 5. 오케스트레이션 (Agentic Orchestration)

Agentic orchestration은 로봇 시스템이 “한 번의 정책 출력”을 넘어 **작업을 쪼개고**, **도구를 호출**하고, **실패를 복구**하는 레이어입니다.

### 무엇을 오케스트레이션하나

* 태스크 플로우: pick → place → 검사 → 보고.
* 도구 호출: 비전 검사, 경로 계획, 작업 DB 조회.
* 휴먼 인 더 루프: 승인/중지/재시도.
* 관측/복구: 타임아웃, 재시도, 폴백 정책.

### 아키텍처 원칙

* 로봇 제어 루프(실시간)와 오케스트레이션(비실시간)을 분리합니다.
* 오케스트레이션은 상태를 남깁니다.
* 권한과 감사 로그가 기본입니다.
