# 로봇 데이터 수집 파이프라인

로봇 데이터 수집은 **대역폭**과 **신뢰성**이 이슈입니다. 모든 원천을 실시간 업로드할 필요는 없습니다.

### 권장 패턴

* 현장에서 1차 버퍼링을 합니다.
* 이벤트 기반으로 업로드합니다.
* 중요 구간만 잘라서 올립니다.
* 업로드 실패 시 재시도합니다.

### 데이터 타입별 팁

* 비디오: 구간 분할(예: 1\~5분) + 해시로 무결성 확인.
* 텔레메트리: 시간 동기화가 우선입니다.
* 안전 이벤트: 가장 먼저 업로드해야 합니다.

### AWS 구성 예시(요약)

* 디바이스 연결/인증: AWS IoT Core.
* 스트리밍 수집: Amazon Kinesis.
* 저장: Amazon S3.
* 이벤트/파이프라인 트리거: Amazon EventBridge.
