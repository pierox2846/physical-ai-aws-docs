# 1. 데이터 (Data)

Physical AI에서 데이터는 보통 **비디오 + 상태 + 이벤트** 조합입니다. 데이터가 커서, "나중에 학습"을 전제로 설계해야 합니다.

### 데이터 소스

* 로봇 텔레메트리: joint, pose, force/torque.
* 비전: RGB, depth, segmentation.
* 이벤트: 실패 코드, 안전 정지, 작업 라벨.
* 시뮬: 시나리오/랜덤 시드/정답 라벨.

### 아키텍처 원칙

* 원천 데이터는 **변형 없이** 장기 보관합니다.
* 메타데이터를 반드시 같이 저장합니다.
  * 시간 기준: 센서 동기화 타임스탬프
  * 로봇 ID, 펌웨어/캘리브레이션 버전
  * 태스크 정의와 성공 기준
* 태스크/로봇/환경 축으로 검색 가능해야 합니다.
* 재현을 위해 데이터와 코드 버전을 같이 남깁니다.
